# Contributing

Thanks for helping improve this repository. Please follow these steps when contributing resources or changes.

How to contribute
1. Fork the repository.
2. Create a descriptive branch from main, e.g. `add-45nm-links` or `improve-quickstart`.
3. Add or update files (edit PDK_RESOURCES.md, add examples, improve docs).
4. Commit with a descriptive message and open a Pull Request against `kured-pdk/ProcessDevelopmentKit:main`.
5. In the PR description, explain the change, link to any external references, and mention if any follow-up work (CI, tests) is required.

Style and content
- Prefer short, clear descriptions with links to authoritative sources (vendor docs, academic papers).
- Group PDK entries by node (e.g., 90nm, 65nm, 45nm, 28nm) and type (CMOS, photonics).
- Avoid posting proprietary files; link to vendor portals or publicly redistributable artifacts.

Repo governance
- Maintainers will review PRs and may request changes.
- Major changes may be staged behind branches or discussed in issues first.

Need help?
- Open an issue describing what you want to add and the maintainer(s) can advise on placement and format.
